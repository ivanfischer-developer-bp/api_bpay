================================================================================
  💻 CÓMO USAR EN TUS CONTROLLERS
================================================================================

  ANTES (Mail::to):
  ─────────────────
  Mail::to($email)->send(new NotificacionEmail($asunto, $datos));

  DESPUÉS (Con Trait):
  ───────────────────
  class MiController extends Controller {
      use SendsEmailsTrait;  // Agregar esto
      
      public function metodo() {
          $this->sendEmail($email, new NotificacionEmail($asunto, $datos));
      }
  }

  BENEFICIOS:
  ✓ Automáticamente intenta Microsoft Graph primero
  ✓ Si falla, automáticamente usa SMTP
  ✓ Reutilizable en toda la aplicación
  ✓ Cambiar drivers solo en .env

================================================================================
  🔑 VARIABLES NECESARIAS EN .env
================================================================================

  MAIL_USE_MICROSOFT_GRAPH=false
  MSGRAPH_CLIENT_ID=xxxxx
  MSGRAPH_CLIENT_SECRET=xxxxx
  MSGRAPH_TENANT_ID=xxxxx
  MSGRAPH_USER_EMAIL=notificaciones@empresa.onmicrosoft.com
  MSGRAPH_REDIRECT_URI=http://localhost:8000/api/msgraph/auth/callback

================================================================================
  📊 COMPARACIÓN: ANTES vs DESPUÉS
================================================================================

  ANTES (Mail::to directo):
  ├─ ❌ Si SMTP falla, email se pierde
  ├─ ❌ Sin fallback automático
  ├─ ❌ Difícil cambiar driver
  └─ ❌ Código repetido en 7 controllers

  DESPUÉS (Con Trait + Fallback):
  ├─ ✅ Si Graph falla, automáticamente SMTP
  ├─ ✅ Fallback automático integrado
  ├─ ✅ Cambiar driver en .env
  └─ ✅ Un trait para 7 controllers
